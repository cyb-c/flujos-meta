## 1. Lo que he entendido del contenido

El documento describe un sistema de automatización para transformar documentación técnica en PDF (productos de protección, control y alarma de incendios) en productos publicados en WooCommerce.

### Arquitectura general

- **Web-App externa (WA)**: Actúa como centro de control del proceso, independiente de WordPress
- **WordPress**: Rol de almacenamiento (texto extraído, imágenes en biblioteca de medios)
- **WooCommerce**: Destino final de creación/publicación de productos vía API
- **IA**: Múltiples ejecuciones con roles diferenciados para interpretar y transformar el contenido

### Flujo principal

```
Login WA → Subir PDF → Extraer texto → Procesar con IA → 
Formulario de revisión → Aprobación/Rechazo → API WooCommerce → Publicación
```

### Puntos clave identificados

| Área | Entendimiento |
|------|---------------|
| **Relación PDF-Producto** | 1 PDF → 1 producto (con excepciones: varios modelos o varios productos) |
| **Revisión humana** | Única y decisiva, ocurre en WA antes de API-WooCommerce |
| **Cancelación** | Disponible durante todo el proceso automatizado, requiere confirmación |
| **Gestión de errores** | Log completo desde inicio a fin; admin revisa fallos post-aprobación |
| **Usuarios** | Login validado contra WordPress; Admin = rol admin en WP; Usuario = rol > suscriptor |
| **Almacenamiento** | Subcarpetas en `DIR_ALMACEN_PDF` con nombre del PDF; conservación depende del resultado |

### Estados del proceso respecto a archivos

| Resultado | Tratamiento de subcarpeta |
|-----------|---------------------------|
| Cancelación confirmada | Se borra todo salvo log |
| Rechazo/desaprobación | Se borra todo con log de descarte |
| Éxito en WooCommerce | Se conserva todo |
| Fallo post-aprobación | Se conserva todo para revisión admin |

## 4. Preguntas de aclaración

Antes de proceder a crear el Product Vision Document, necesito aclarar los siguientes puntos:

### Sobre objetivos y alcance

1. **¿Cuál es el objetivo de negocio principal?** ¿Reducir tiempo de publicación? ¿Eliminar errores manuales? ¿Escalar el volumen de productos publicables?

2. **¿Existe un volumen objetivo de procesamiento?** ¿Cuántos PDFs se espera procesar por día/semana/mes?

3. **¿Hay un plazo o fecha objetivo para la implementación?**

### Sobre la Web-App

4. **¿La WA debe ser desarrollada desde cero o existe alguna base tecnológica preferida?** (framework, lenguaje, hosting)

5. **¿La WA debe alojarse en el mismo servidor que WordPress o puede estar en un entorno separado?**

6. **¿Existe alguna restricción tecnológica para la WA?** (PHP, Node.js, Python, etc.)

### Sobre la IA

7. **¿El "documento dedicado a IA" mencionado existe o está pendiente de crear?**

8. **¿Hay algún proveedor de IA ya seleccionado o presupuestado?** (OpenAI, Anthropic, local, etc.)

9. **¿Los "roles de IA" implican llamadas separadas a la API o son prompts diferentes dentro de la misma sesión?**

### Sobre WordPress/WooCommerce

10. **¿Existe ya un sitio WordPress/WooCommerce en producción o es parte del mismo proyecto?**

11. **¿Hay campos personalizados de producto en WooCommerce que deban considerarse?**

12. **¿La tabla específica en WordPress mencionada en el boceto ya existe o debe crearse?**

### Sobre usuarios

13. **¿Los usuarios de la WA son los mismos que los de WordPress o hay un subconjunto específico?**

14. **¿Se requiere algún tipo de auditoría o trazabilidad de quién aprobó cada producto?** (más allá del log técnico)

### Sobre el documento resultante

15. **¿El PVD debe ser un documento técnico para desarrolladores o un documento de negocio para stakeholders?** ¿O ambos en secciones diferenciadas?

16. **¿Hay alguna plantilla o formato estándar que deba seguirse en la organización?**

17. **¿Qué nivel de detalle se espera en la representación gráfica del flujo?** (diagrama ASCII, mermaid, descripción textual detallada)
