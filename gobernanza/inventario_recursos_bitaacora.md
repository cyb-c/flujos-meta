## Reglas de Registro

1. Todo cambio en `inventario_recursos.md` debe registrarse en esta bitácora.
2. Cada registro debe incluir: fecha, descripción del cambio, responsable y aprobación.
3. Los cambios se registran después de ser implementados y verificados.
4. No se registran cambios hipotéticos o pendientes.
5. Esta bitácora es el único lugar donde se mantiene el historial de cambios del inventario.

## Plantilla de Registro

Para registrar un nuevo cambio, usar la siguiente fila:

```
| YYYY-MM-DD | [Descripción del cambio] | [Nombre] | [Nombre] |
```

### Tipos de Cambio

| Tipo | Descripción |
|------|-------------|
| Creación | Nuevo recurso, variable o configuración añadida |
| Modificación | Cambio en valor, descripción o configuración existente |
| Eliminación | Recurso, variable o configuración removido |
| Corrección | Error corregido en entrada existente |
| Verificación | Estado verificado o actualizado |

---

> **Nota:** Esta bitácora es complementaria a `inventario_recursos.md`. El historial de cambios no debe estar dentro del inventario principal.
